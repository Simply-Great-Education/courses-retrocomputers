v5 - 2024-01-06
---------------

New Features or Changes
-----------------------

Reworked the cartridge boot code for better stability and tweaked the loading
raster bar simulation for a more authentic look.

Assault Machine and Castle of Terror now have working MultiLoad64 conversions,
so I've finally started to help with this part of the collection! I certainly
plan to be more active in this area in upcoming releases. :)

$olo1870 has been busy adding many more games to MultiLoad64 himself, which is
great to see. Many thanks to him and all those giving him suggestions for games
to add, as well as testing his conversions. He has also tweaked many of his
conversions such that the games now load much quicker and have extras such as
intros etc. He even managed to produce an 11th hour fix for Tintin on the Moon.
Awesome stuff!

hellboy73 has also added Gumball, Hostages and Muncher Eats Chewits to
MultiLoad64. All excellent additions I'm sure you'll agree.

And thanks to redmist77, we not only have a working conversion of HardBall! in
the collection now, but also conversions of all the Leaderboard games. Very
nice work indeed.

Games Fixed or Improved
-----------------------

1942
    - As the re-released version on the Encore label has different music to the
      original Elite release, this is now also included. Thanks to nokou for
      the suggestion.

4th and Inches
    - The game uses an arbitrary multiload protection after each match quarter
      which needed to be bypassed. Thanks to redmist77 for creating the fix, as
      well as adding the loading screen from the disk version to the CRT.

Adrenalin
    - This has now been fixed such that it does not reload the intro from disk
      on game over. Thanks to jc70 for the issue report.

Alleykat
    - The CRT now contains the loading screen by Lizard (LIZ) which was ripped
      from the version in GameBase64. Thanks to leventp, ricky006 and
      Wayne / Art Ravers for sharing info for this on the Lemon64 forums.

Aquaplane
    - This was incorrectly labelled as a joystick port 1 game, but in fact uses
      the joystick in port 2 if 'Q' is pressed on the title screen. The game is
      now patched so that pressing the fire button in port 2 on the title screen
      enables joystick control mode and starts the game. Thanks to tiktokofoz
      for the issue report.

Ardy The Aardvark
    - The CRT now contains a conversion of the loading screen from the Atari
      8-bit computer version.

Assault Machine (MultiLoad64)
    - Has now been correctly converted and added to MultiLoad64 as it did not
      correctly support the loading of the equip droids menu. Thanks to jc70
      for the issue report.

Battle Ships
    - Fixed an issue which caused the vessels not to be placed randomly. Thanks
      to kamil8 for the issue report.

Castle of Terror (MultiLoad64)
    - Has now been correctly converted and added to MultiLoad64 (it was
      removed in the last version as it did not correctly support the loading
      of the second part). Thanks to jc70 for the issue report.

Cyberdyne Warrior
    - Fixed an issue which caused corrupted graphics after completing the 2nd
      level. Thanks to mmhh for the issue report.

Cybernoid
    - The Spectrum loading screen used for the CRT has been replaced with the
      one by Stephen Robertson (SIR) obtained from CSDb.

Davy - King of the Wild Frontier
    - Now uses a dump from side B of the tape which uses a fastloader, as this
      has correct in-game speech, whereas the kernal loader on side A does not.
      Furthermore, low memory is now also initialised before starting the game
      to remove further issues with the speech. Thanks to Zombeaver for the
      issue report.

Delta
    - The Spectrum loading screen used for the CRT has been replaced with the
      one by Lizard (LIZ) obtained from CSDb.

Driller
    - Removed the manual protection when loading a saved game (this was already
      removed when starting the game). Thanks to Zombeaver for the issue
      report.

Dropzone
    - The CRT now contains the updated loading screen by Steve Day (STE'86)
      obtained from CSDb.

Eagle Empire
    - Fixed an issue which prevented the ship shield sprite animation to not
      work correctly with the CRT version. Thanks to ricky006 for the issue
      report.

Gunfright
    - Fixed an issue which prevented forward movement of the character with any
      control method. Thanks to FaberfoX for the issue report.

Harvey Smith Showjumper
    - Fixed an issue which caused the game to crash when the first round
      finishes. Thanks to woodlandcreature74 for the issue report.

Livingstone I Presume
    - Fixed an issue which caused the game to crash part way through the game.
      The Novaload .TAP file in Dumps\SourceFiles has also been fixed. Thanks
      to Robin from the 8-Bit Show and Tell YouTube channel for the fix. The
      CRT now also contains the Spectrum loading screen.

Madballs
    - Fixed an issue which caused the letter selection cursor on the high score
      name entry screen to be corrupted. Thanks to Zombeaver for the issue
      report.

Micro Ball
    - The CRT now plays the loading music at the faster, correct speed (i.e.
      it's now synced against CIA#1 timer A at 60Hz, instead of the raster
      interrupt at 50Hz on a PAL C64).

Mission Impossibubble
    - Now initialises colour RAM so that the game displays correctly. Thanks to
      redmist77 for the issue support.

Nodes of Yesod
    - As the version initially in the collection was the faster version from
      side B of the original tape, the slower version from side A has also
      been added. The game is now also patched so that it defaults to joystick
      control and pressing the fire button in port 2 starts the game. Thanks to
      Zombeaver for the issue report.

Pancho
    - The game is now patched so that pressing the space bar or the fire button
      in port 2 at any time on the title screen starts the game. Thanks to
      Zombeaver for the suggestion.

Parallax
    - Fixed an issue which caused the title screen music to play incorrectly.
      This was caused by a bad memory dump ($A4C7 was $85 instead of $8D).
      Thanks to lmforg and ricky006 for the issue report.

Poster Paster
    - Fixed an issue which prevented the options screen from being displayed for
      the correct amount of time before returning to the title screen. Thanks to
      Mousey for the issue report.

Raid on Bungeling Bay
    - Fixed an issue which caused the helicopter rotors and engine sound to not
      gradually spin up to speed when taking off for the first time. Thanks to
      tiktokofoz for the issue report.

Raid over Moscow
    - Fixed an issue which caused corrupted graphics at the bottom of the world
      overview map. Thanks to tiktokofoz for the issue report.

Ricochet
    - Fixed an issue which caused there to be no in-game sound. Thanks to
      Wayne / Art Ravers for the issue report.

Run for Gold & Scare Bear
    - The CRTs now play the loading music at the faster, correct speed (i.e.
      it's now synced against CIA#1 timer A at 60Hz, instead of the raster
      interrupt at 50Hz on a PAL C64).

Speedball
    - Fixed an issue where the use of certain IO registers in the loading
      screen display code of the CRT version caused the match part of the game
      to run very slowly. Thanks to Bovirtual for the issue report.

Star Paws
    - Fixed an issue which caused corrupted graphics in the mine sections. The
      issue was caused by the fact that the tape dump used did not allow for a
      devious copy protection mechanism which checks how much blank tape is left
      after the game has finished loading. Therefore, we are now using a fixed
      version which has been trimmed to match the tape length of the original.
      Thanks to pistolhamster for the issue report, as well as PonPon and the
      Ultimate Tape Archive team for providing the fixed TAP image.

Starglider
    - Fixed so that the music/sound begins as soon as the game starts just like
      the disk version. Thanks to tiktokofoz for the issue report.

Strangeloop
    - Fixed an issue which caused the game to crash when trying to display the
      dissolved message after coming into contact with acid pools. Thanks to
      Mousey for the issue report.

Thing on a Spring
    - The CRT now contains the loading screen by Stephen Robertson (SIR)
      obtained from CSDb.

Titanic Blinky
    - Fixed an issue which prevented the character from shooting anything or
      losing any energy. Thanks to kamil8 for the issue report.

Tintin on the Moon (MultiLoad64)
    - Fixed an issue which prevented the completion screen from displaying at
      the end of the game. It seems this bug exists in the original tape and
      disk versions. Thanks to PonPon and hayesmaker64 for the issue report.

Track and Field
    - Removed some copy protection which caused the game to crash on game over.
      Thanks to CapFuture1975 for the issue report.

Uridium
    - The Spectrum loading screen used for the CRT has been replaced with the
      one by Steve Day (STE'86) obtained from CSDb.

v4 - 2022-12-25
---------------

New Features or Changes
-----------------------

Given that the root folder was becoming rather cluttered, its structure has been
simplified and some folders have been renamed for better clarity. Here are the
old to new path mappings:

    BINs --> AlternativeFormats\BINs
    EasyFlash --> AlternativeFormats\EasyFlash
    PRGs --> AlternativeFormats\PRGs
    Tapecart --> AlternativeFormats\Tapecart
    Originals --> Dumps\SourceFiles
    Screenshots --> Extras\Images\Screenshots
    Extras\OriginalCRTs --> Extras\OfficialCRTs

100 multiload game conversions are now included in the MultiLoad64 folder. Many
thanks to $olo1870 and hellboy73 for their hard work to put this first batch of
games together. As we add more multi-load games to future versions of the
collection, they will be included here.

The AlternativeFormats\BINs folder now contains a further subfolder containing
BIN files for use with Hucky (RGCD) cartridges. As these are limited to 64kb,
they do not contain any added extras such as loading screens or music.

Loading screens for games which have them have been added to the
Extras\Images\LoadingScreens folder.

Thumbnail images for the Sidekick64 device have been added to the
Extras\Images\Sidekick64Thumbnails folder in TARGA (TGA) format. This includes
an additional file in slideshow format where the game also has a loading screen.
These should be placed in the same folder as the CRT files for the Sidekick64 to
use them. Thanks to Frenetic for the addition of the slidehow feature to the
Sidekick64 firmware and his general assistance and advice.

Where a game was released after the year 2000, this info has been added to the
filename to allow more recent games to be identified more easily. Thanks to
Snarko for the suggestion.

The TAP image used for this game has been fixed, as the data in the file between
$3698A-$4C271 (which is loaded to $54E2-$7FFF in RAM) is corrupted. Therefore
this data was replaced with a dump of this same data from RAM using the T64

v3 - 2022-02-26
---------------

New Features or Changes
-----------------------

Added support for loading music and raster bars to the Magic Desk CRTs which
are combined with loading screens for a simulated tape loading experience
(these can be skipped with any key or either fire button just like the plain
loading screen images). This means that games with loading music such as the
classic Ocean loaders can now be enjoyed with the CRT files.

The MagicDeskBINs folder now contains two subfolders, one which contains plain
64kb files, along with another which contains larger files where games have
added extras like loading screens or music. This allows more flexibility when
choosing which EPROM size to use when writing the games to a physical
cartridge.

Games using joystick port 1 for primary control are now denoted by having the
(J1) suffix in their filename. This not only acts as a visual cue to the user,
but is also automatically used by emulators such as LR-VICE to swap joystick
ports (https://github.com/libretro/vice-libretro#joyport-control).

All screenshots have now been replaced with new images at 384x272 to ensure
the borders are included. Many thanks to the volunteers who helped to perform
this task (CentralPerk, CommFor, cringle, docbobo, nls, nc513, oldschooljoe,
Remzi and ricky006).

The crunched PRGs no longer have the "-[ex]" suffix in their filename, as
these were rather redundant and made the process of maintaining the collection
a little harder than it needed to be (i.e. when renaming files in conjunction
with the new primary control suffix above).

Added the use of $D030 to switch to 2Mhz mode on the C128 and the MiSTer C64
core to speed up the decrunch of the crunched PRGs (has no effect on a real
C64).

      called from multiple places in the game code). Thanks to Espressomatic for
      the issue report.

New Features or Changes
-----------------------

The primary uncrunched games are now in Magic Desk CRT format for even quicker
loading. These also support the display of loading screens which wasn't always
possible with the original PRG format due to space limitations. The uncrunched
PRGs are still included for use when Magic Desk cartridge support is not
available on a specific platform or device.

Screenshots are now included for all the core games in the collection. Future
work is planned to improve these by replacing with larger resolution files
which capture the full C64 screen, including the borders which are used by some
games. But for now, they are the traditional 320x200.

Official cartridge images are now included in Extras\OfficialCRTs
(thanks to Mayhem for his assistance in putting this set together)

Other cartridge images of interest (e.g. multiload games converted to CRT
format etc) are now included in Extras\OtherCRTs
